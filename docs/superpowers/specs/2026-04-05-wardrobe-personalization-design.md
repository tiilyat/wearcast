# Персонализация рекомендаций одежды через гардероб пользователя

**Дата:** 2026-04-05
**Статус:** Утверждён

## Проблема

AI рекомендует специализированную одежду (термобельё, мембранные куртки и т.д.), предполагая что она есть у пользователя. Это делает рекомендации непрактичными для многих пользователей.

## Решение

Двухуровневый подход:

1. **Базовый:** улучшить промпт — по умолчанию рекомендовать только общедоступную повседневную одежду
2. **Расширенный:** опциональная настройка гардероба — пользователь отмечает что у него есть, AI строит рекомендации только из этих вещей

## Изменения в промпте

### Без гардероба (по умолчанию)

Добавить в системный промпт:

```
- Рекомендуй только общедоступную, повседневную одежду: футболки, рубашки, свитера, джинсы, куртки, кроссовки и т.п.
- Не предлагай специализированную одежду (термобельё, мембранные куртки, горнолыжное снаряжение), если это не экстремальные условия
- Если погода требует специализированной защиты — объясни это в warnings, но основная рекомендация должна быть из обычных вещей
```

### С гардеробом

Добавить в user message:

```
Гардероб пользователя:
Базовый слой: футболка, термобельё, лонгслив
Средний слой: свитер, худи
Верхний слой: пуховик, ветровка
Аксессуары: шапка, перчатки

Рекомендуй только из этих вещей. Если чего-то критичного не хватает — укажи в warnings.
```

### Хранение

- **localStorage**, ключ `wearcast-wardrobe`
- Формат: `Record<LayerType, string[]>` (где LayerType = "base" | "mid" | "outer" | "accessory")
- Пустой/null = гардероб не настроен → используется базовый промпт
- Zod-схема `Wardrobe` в `src/lib/schemas.ts`

## UI: Drawer настройки гардероба

### Точка входа

Кнопка с иконкой `Shirt` (lucide) рядом с селектором активности. Текст: «Мой гардероб». Если гардероб настроен — индикатор (точка или бейдж с количеством вещей).

### Drawer (Sheet)

- Компонент: shadcn Sheet (снизу)
- Заголовок: «Мой гардероб»
- Подзаголовок: «Отметьте вещи, которые у вас есть — рекомендации будут точнее»
- 4 секции-аккордеона по слоям: «Базовый слой», «Средний слой», «Верхний слой», «Аксессуары»
- Внутри каждой секции — чекбоксы с названиями вещей
- Кнопка «Сохранить» — сохраняет в localStorage, закрывает drawer
- Кнопка «Сбросить» — очищает гардероб (возврат к базовому промпту)

### Поведение

- При открытии — читаем состояние из localStorage
- При сохранении — пишем в localStorage, триггерим перегенерацию рекомендации (если она уже была)
- Закрытие без сохранения — изменения не применяются

### Компоненты shadcn/ui

Sheet, Accordion, Checkbox, Button

## Поток данных

```
[Пользователь] → drawer → отмечает вещи → «Сохранить» → localStorage

[Запрос рекомендации]:
  1. Читаем гардероб из localStorage
  2. Передаём в getOutfitRecommendation как параметр wardrobe
  3. Серверная функция формирует промпт:
     - wardrobe пустой → базовый промпт «только общедоступная одежда»
     - wardrobe заполнен → секция «Гардероб пользователя: ...» в user message
  4. AI генерирует рекомендацию → клиент
```
